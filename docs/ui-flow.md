# UI Flow

## Последовательность экранов

1. **Protocol selection**
   - Выбор протокола эксперимента.
   - Основные действия:
     - MainButton: перейти к сборке плана (Plan builder).
     - BackButton: вернуться на предыдущий экран приложения.
2. **Plan builder**
   - Формирование шагов эксперимента и параметров протокола.
   - Основные действия:
     - MainButton: перейти к проверочному списку (Checklist).
     - BackButton: вернуться к выбору протокола.
3. **Checklist**
   - Проверка готовности: материалы, реагенты, оборудование.
   - Основные действия:
     - MainButton: перейти к загрузкам (Uploads).
     - BackButton: вернуться к сборке плана.
4. **Uploads**
   - Загрузка файлов, фото и данных результатов.
   - Основные действия:
     - MainButton: перейти к итоговому экрану (Summary).
     - BackButton: вернуться к чек-листу.
5. **Summary**
   - Итоговый обзор эксперимента, подтверждение завершения.
   - Основные действия:
     - MainButton: отметить эксперимент как завершенный.
     - BackButton: вернуться к загрузкам.

## Состояния

- **Черновик**: создан выбор протокола и/или план, но эксперимент еще не запущен.
- **В работе**: выполнен чек-лист или начаты загрузки, эксперимент выполняется.
- **Завершено**: заполнены загрузки и подтвержден итоговый экран.

## Ключевые действия

- **MainButton** используется для перехода к следующему шагу и подтверждения итогового экрана.
- **BackButton** используется для возврата на предыдущий экран без потери введенных данных.
